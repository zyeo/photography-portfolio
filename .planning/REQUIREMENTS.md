# Requirements: Photography Journal & Portfolio

**Defined:** 2026-05-15
**Core Value:** A visitor can encounter Zach's visual voice immediately, then follow the daily discipline that produces it.

## v1 Requirements

### Foundation

- [x] **FOUND-01**: Public application shell runs locally with responsive navigation and layout primitives.
- [x] **FOUND-02**: Visual system expresses the approved paper-white, charcoal, poetic-minimal direction.
- [x] **FOUND-03**: Homepage supports contrast-safe hero presentation behavior.

### Data & Media

- [x] **DATA-01**: System stores photos, journal entries, collections, and photo-collection relationships.
- [x] **DATA-02**: Original files remain private while public image delivery assets can be served efficiently.
- [x] **DATA-03**: Uploads extract EXIF where available without requiring metadata to exist.
- [x] **DATA-04**: Hero focal point and mobile crop preferences can be stored.

### Admin

- [x] **ADMIN-01**: Admin user can authenticate into a private admin area.
- [ ] **ADMIN-02**: Admin can publish one daily journal entry end to end.
- [ ] **ADMIN-03**: Admin can upload archive photos in batches with optional shared metadata and per-photo review.
- [ ] **ADMIN-04**: Admin can search, filter, and edit library records.
- [ ] **ADMIN-05**: Admin can curate Selected work, collections, and homepage hero behavior.

### Public Experience

- [ ] **PUBLIC-01**: Visitor can browse an immersive homepage with approved hero imagery.
- [ ] **PUBLIC-02**: Visitor can browse Selected work in an editorial gallery and lightbox.
- [ ] **PUBLIC-03**: Visitor can browse a reverse-chronological journal and individual image-first entries.
- [ ] **PUBLIC-04**: Visitor can browse collections and collection detail pages.
- [ ] **PUBLIC-05**: Visitor can read an About page with quiet contact availability.

### Launch Readiness

- [ ] **LAUNCH-01**: Site includes SEO basics, sitemap, robots, and social metadata.
- [ ] **LAUNCH-02**: Site handles loading, empty, and error states accessibly.
- [ ] **LAUNCH-03**: Deployment and operator documentation are reproducible.

## v2 Requirements

### Future Expansion

- **FUTURE-01**: Visitor can browse by place or neighborhood.
- **FUTURE-02**: Visitor can open dedicated public photo URLs from lightboxes.
- **FUTURE-03**: Site supports richer essays, services, and booking flows.

## Out of Scope

| Feature | Reason |
|---------|--------|
| Booking system | Services are deferred until the offering is concrete |
| Commerce | Not core to the v1 journal/portfolio value |
| Places section | Archive needs to earn the feature first |
| Decorative motion-heavy UI | Conflicts with the calm, image-led direction |
